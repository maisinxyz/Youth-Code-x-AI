# Engram — Design Context

## Target audience

Engineers, PMs, and founders at technical companies. Power users of developer tools. They expect tools that respect their attention and feel like infrastructure, not consumer products.

## Use cases

Engram is the institutional memory layer for an organization. Users connect their data sources (Slack, Notion, Drive, Confluence, Jira, Teams), watch a 3D semantic graph build itself from that data, and then query it in natural language — voice or text — to surface answers grounded in real source material.

The atomic loop: **ask the brain a question → get a grounded answer with traceable sources.**

## Brand personality / tone

- **Cinematic, not playful.** The brain visual is the hero. Restraint is a feature.
- **Jarvis-like intelligence.** Quiet capability. Nothing is shouted. Confidence without ornament.
- **Infrastructure, not toy.** Feels closer to a terminal or observatory than a dashboard or app.
- **Voice is declarative.** No hedging copy ("we think", "maybe try"). No exclamation points. No emoji.

## Design language

- **Theme:** Dark only. Near-black backgrounds with thin layering. Pure black `oklch(0% 0 0)` reserved for the 3D canvas.
- **Palette:** Achromatic black/grey/white base. Forest green `#2d6a4f` (`oklch(42% 0.095 162)`) is the **only** chromatic accent. Used sparingly — for activation, focus, primary CTA, the cascade lighting up the graph.
- **Typography:**
  - Display: **Barlow Condensed** (700, 800) — architectural, narrow, cinematic at large scale
  - Body: **Geist Mono** (400, 500) — clean monospaced, signals "tool not toy"
- **Spacing:** 4pt scale. Vary spacing for hierarchy; don't apply the same padding everywhere.
- **Motion:** All enters use `ease-out` cubic-bezier `(0, 0, 0.2, 1)`. No bounce. No elastic. Hover transitions ≤150ms. Section reveals 600ms. Cinematic camera drifts 1200ms. `:active` scale(0.97) on every interactive element.

## Anti-patterns (never do)

- Rainbows or multi-color gradients
- Gradient text fills (BAN: `background-clip: text` with gradient)
- Side-stripe colored borders on cards (BAN: `border-left: Npx solid <color>` >1px)
- Glassmorphism for decoration (only when functional, e.g. floating panels over the brain)
- Emoji in UI
- Friendly chatbot personas
- Marketing fluff copy
- Bright neon-on-dark "AI palette"
- Generic rounded-rectangle drop-shadow card grids
- Modals (use drawers, panels, or inline reveals instead)

## Tokens

All colors, spacing, typography, animation, radii, and shadows are in `design/tokens.json`. Frontend imports them via `tailwind.config.ts`. **Never** hardcode hex/rgb/oklch in components — use token-mapped Tailwind classes.
